JSP_EL_JSTL的学习(一)

2017-09-26

JSP_EL_JSTL的学习

  • JSP（JavaServer Pages）, 是一种可用Java和Html结合的动态页面，解决了Java中的Servlet繁琐的输出语句造成的繁重的工作量。

Jsp可以被看做是Servlet，事实上，在JSP文件第一次被运行的时候，会被当前的Web容器翻译成Servlet之后才会运行。

第一次访问之后，Web容器会将.jsp结尾的文件转换为原文件名加下划线（_）之后加jsp的以java为后缀名的文件， 例如Hello.jsp会被转换为Hello_jsp.java,之后再运行

JSP_Syntax

  1. <% Java代码 %>

    第一种方式用于定义方法体内部的局部变量和一些执行步骤

  2. <%= java变量或者表达式 %>

第二种方式用于向网页输出变量的值或者表达式的结果

  1. <%! Java代码 %>

第三种方式可以定义类中的方法和成员变量，其中的代码会被转换成类的成员

JSP的注释

  1. Html一样的注释：<!--注释--> JSP文件，翻译后的Servlet，页面显示源代码都可见

  2. Java的注释：//单行注释 /*多行注释*/ Jsp源文件，翻译后的Servlet文件

  3. JSP的注释：<%--注释--%> 之后Jsp源文件可见

转换成servlet文件之后

JSP指令

  • page

用于描述页面的属性，例如页面的编码方式，jsp页面采用的语言等等。

<!--page指令使用格式-->
<%@page 属性1="属性1" 属性2="属性2" 属性3="属性3"...%>

Page用来声明指令名称，属性用来指定jsp页面的某些特性。

pageContext对象：

JSP页面的环境变量对象（上下文对象），用于访问jsp页面中所有对象以及命名空间。

pageContext是一个域对象

域对象同意拥有并且多次使用的方法:

  • setAttribute(String name, Object obj)
  • getAttribute(String name)
  • removeAttribute(String name)

pageContext提供了获取其他隐式对象的方法：

  • JSPWriter getOut()
  • Object getPage()
  • ServletRequest getRequest()
  • ServletResponse getResponse()
  • HttpSession getSession()
  • HttpException getException()
  • ServletConfig getServletConfig
  • ServletContext getServletContext() 用于获得application内置对象

pageContext操作属性的方法:

  • setAttribute(String name, Object obj, int scope) 设置指定隐式对象中的属性
  • getAttribute(String name, Object obj, int scope) 获取指定隐式对象中的属性
  • removeAttribute(String name, Object obj, int scope) 删除指定范围中的属性
  • findAttribute(String name) 从4个域对象中查找对应名称为name的属性，根据域的范围从小到大搜索，分别是pageContext < request < session < ServletContext

##动态页面包含和静态页面包含的区别(重点记忆)

  • 静态包含<%@include file=”被包含页面的相对地址”%>

    当前页面和被包含页面合并之后被编译成一个Servlet源码和字节

    两个页面的<%@page contentType=”” charset=””%>必须一致

    包含的jsp页面和被包含的jsp页面共用一个request内置对象

    被包含的jsp页面和包含的jsp页面不应该有相同的http对象，否则会被覆盖

  • 动态包含

    当前页面和被包含页面分别编译成一个Servlet源码和字节，运行时，由tomcat进行合并且输出

    两个页面里，被包含页面拥有自己的request内置对象，同时可以访问包含页面接受的request的内置对象，所以被包含页面的参数大于等于包含页面的参数

//第一种传参方式
    <jsp:include page="../asp.jsp?param=123"/>
//第二种传参方式
    <jsp:include page="../asp.jsp">
        <jsp:param name="" value="" >
        <jsp:param name="" value="" >
    </jsp:include>

Xml – Extensible Markup Language

XML（可拓展性标记语言）：XML中的标签是可以自己定义的，同样属性也是可以自定义的，具有极大的自由度，XML被设计为存储和传输数据，其焦点是数据的内容。

但是XML对于自定义的标签有一定的要求：

  • XML对于大小写敏感
  • 每个XML标签都必须要有结束标签，例如<head></head>
  • XML必须正确的嵌套
  • XML必须有根元素
  • XML标签中的值必须用“”括起来

    • XML中的实体引用

    在XML中一些字符拥有特殊的含义，例如

    如果直接把这些符号放入到xml文本中去就会出现错误，为了避免这种现象的发生，采用实体引用代替该字符。

    XML最常见的应用场景：

        1. 配置文件
        2. 存放数据(目前使用最多的是Json)

    XML语法

    1. 文档声明

    <?xml version=”1.0” encoding=”UTF-8”>

    a. 文档开头必须以<?开头，>?结尾;

    b. 文档声明必须从文档的第0行0列位置开始

    c. 文档声明只有三个属性：

    1. version 指定xml文档版本，一般使用1.0
    2. encoding 指定当前文档的编码格式，默认为UTF-8
    3. standalone 是否需要依赖其他文件，yes/no

    2. XML元素 （ELement）

    1. 元素是XML文件中最主要的组成部分

    2. 普通元素的结构由开始标签，元素体，结束标签组成，例如：

      <hello> This is content </hello>

    3. 元素体（元素体可以是文本，也可以是元素），例如：

      <b><a>content</a></b>

    4. 空元素 空元素只有开始标签没有结束标签，元素必须自己闭合。例如<br />

    5. 元素命名

      a. 区分大小写
      b. 不能使用空格
      c. 不建议以XML，xml，Xml等字符为开头

    1. 格式化良好的XML文件只能有一个根元素

    3. XML属性

    1. 属性是元素的一部分，他必须出现在元素开始标签中
    2. 属性定义格式：属性名=属性值，其中属性值用单引号或者双引号括起来
    3. 一个元素可以有0~n个属性，任何一个元素中不能有同名的属性
    4. 属性名不能有空格，冒号等特殊字符，必须以字母开头

      XML注释

    XML转义字符与HTML中的一样

    CDATA区

    当XML文档中出现大量的转义字符时，会影响文档的可阅读性，此事可以使用CDATA区来将这些大量的转义字符包含起来就可以了
    在CDATA区中的< > & ' "都无需转义字符

    常见的XML约束：DTD & Schema

    DTD约束(Document Type Document)

    文档类型定义，用来约束XML文档，规定XML文档中名称，子元素的名称以及顺序，元素的属性等等。

    开发中常见的使用DTD约束的开发框架有Struct2, Hibernate等

    DTD-元素

    元素的声明 :
    <!ELEMENT 元素名称 类别> 或者 <!ELEMENT 元素名称 (元素内容)>

    空元素通过EMPTY关键字进行声明
    <!ELEMENT 元素名称 EMPTY>

    只有PCDATA的元素通过圆括号中的#PCDATA进行声明
    <!ELEMENT 元素名称 (#PCDATA)>
    PCDATA表示标签体中只能包含文本，不能包含子标签

    带有任何内容的元素
    通过类别关键字ANY声明的元素，可包含任何可解析的数据的组合
    <!ELEMENT 元素名称 ANY>

    带有子元素内容的标签
    带有一个或者多个子元素的标签通过元素圆括号中的子元素名进行声明
    <!ELEMENT 元素名称 (子元素名1，子元素名2，子元素名3，....)>

    元素声明中常见的符号及其表示意义
    ? 表示子元素可以出现零次或者一次
    * 表示子元素可以出现零次或者多次
    + 表示子元素可以出现一次或者多次
    | 表示子元素多选一

    DTD-属性

    在DTD中，属性通过ATTLIST关键字来进行声明标签属性
    声明属性使用下列语法：
    <!ATTLIST 元素名称 属性名称 属性类型 约束>

    Schema

    XML Schema是基于XML的DTD的代替者
    XML Schema描述XML文档的结构
    XML Schema语言也被称为XML Schema Defintion, XSD

    常见的使用XML Schema的框架：Spring等

    Java中的网络编程

    1. 网络通信协议

    使用最广泛的网络通信协议是TCP/IP协，包括TCP协议和IP协议, UDP协议和其他的一些协议组成的协议组。

    TCP/IP的层次结构分为四层：


    • 应用层&nbsp&nbsp&nbsp&nbsp(应用程序之间的协议，如：HTTP/FTP协议)

    • 传输层&nbsp&nbsp&nbsp&nbsp(主要使网络程序之间可以通信，在进行网络通信的时候，TCP/UDP协议都可以使用)

    • 网络层&nbsp&nbsp&nbsp&nbsp(是传输协议的核心层，将传输数据分组包装，分发到目标计算机/网络)

    • 链路层&nbsp&nbsp&nbsp&nbsp(针对计算机网络中物理连接设备的驱动协议)

    2. IP地址和端口号

    IP地址：在TCP/IP协议中唯一标识一台计算机的一个标识号。IP地址分为IPV4, IPV6.

    • IPV4： 有4个字节(Byte)组成，一共32位(bit)，共有pow(2, 32)个地址容量(算上全零)，但是目前的IPV4地址已经不能满足网络中越来越多的计算机数量了，所以产生了IPV6.

    • IPV6： 有16个字节(Byte)组成，一共128位(bit), 共有pow(2, 128)个地址容量(算上全零的)。

    通过IP地址可以连接到不同的计算机，但是要访问目标计算机的应用程序的话还需要指定端口号。

    端口(port)：在Internet协议套件中，端口是操作系统中通信的端点。当然这个术语有时候也被用于计算机中的一些物理接口中。不过把它用来称呼软件中的一个特殊的进程或者一种网络类型也是合适的。

    端口号(Port Number): 一个端口总是和一个主机的IP地址和通信的类型相关联。这样就完成了通信会话的目的地或者原始网络地址。每个地址和协议按16为数字表示一个端口。这就被称为端口号。
    通俗地说就是不同计算机之间想要访问对方的应用程序的话就必须在IP地址之后提供其端口号。

    端口号的范围取值：[0, 65536]
    一般分为三种:
    系统端口范围：[0, 1023]
    注册端口号范围是: [1024, 49151]
    动态分配的端口号范围为[49152, 65536] (一般用于短暂的分配给用户使用)

    Port numbers are sometimes seen in web or other uniform resource locators (URLs). By default, HTTP uses port 80 and HTTPS uses port 443, but a URL like http://www.example.com:8080/path/ specifies that the web browser connects instead to port 8080 of the HTTP server.

    127.0.0.1指回环地址，用来表示本机地址，一般用来测试本机，在后面的代码中会用此IP地址测试程序。

    Java中的反射机制

    Java中的反射机制

    某些时候，程序在编译之后需要添加一些功能，但是源代码是不能改变的，所以就需要在不改动源代码智商为程序添加一部分功能， 由此引出了Java的反射机制

    类的加载

    1. 概述：

      Class文件由类加载器装载之后，在JVM中形成一个关于描述Class结构的源信息对象，通过该元素信息对象可以获知该class的结构信息，如构造函数，属性和方法等等。java允许用户借由这个class相关的元信息对象间接调用class对象的成员。

    虚拟机把描述类的数据从class文件加载到内存，并对数据进行校验，转换，解析和初始化，最终形成可以直接被虚拟机直接使用的java类型。这就是虚拟机的类加载机制

    来源

    类加载器（ClassLoader）

    1. Bootstrap Classloader:

      将存放于\lib目录中的或者被-Xbootclasspath参数所指定的路径中，并且是虚拟机识别的(仅按照文件名识别，如rt.jar名字不符合的类库即使放在lib目录中也不会被加载)类库加载到虚拟机内存中，启动类加载无法被java程序直接引用。

    2. Extension Classloader:
      将\lib\ext目录下的，或者被java.ext.dirs系统变量所指定的路径中的所有类库加载，开发者可以直接使用扩展类加载器。

    3. Application Classloader:
      负责加载用户类路径classpath上所指定的类库，开发者可以直接使用。

    反射机制

    前面说到，Java虚拟机允许用户借由class的元信息对象间接引用类的成员，java中的反射机制就是用到了上面说的元信息对象来对程序进行修改的操作。

    也就说反射机制降低了程序对于程序所需要用到的不同类的耦合性，在新的程序中程序可以通过读取程序中的配置文件来决定做什么操作。

    1.获取类的源文件对象有三种方式：

    • Class.forName(String className) 使用Class类的静态方法，传入类的全名(包名.类名)获取类的元文件对象
    • 对象名.getClass() 使用当前对象的getClass()方法获取类的元文件对象
    • 类名.class 直接使用类的隐藏属性class获取类的元文件对象

    1. 根据传入的参数类型寻找对应的的公共权限的构造方法

    1. 根据传入的参数类型寻找对应的的所有权限的构造方法

    1. 获取元文件对象中根据参数找寻对应的成员方法(所有权限)

    1. 获取元文件对象中的公共权限的属性
    1. 指定属性名字从元文件对象中寻找对应的属性(公共属性)
    1. 获取元文件对象中所有声明的属性(权限不限)
    1. 指定属性名在元文件对象中寻找对应的属性(权限不限)

    谈谈Java中的线程

    2017-08-06

    谈谈Java中的线程

    Process (computing)

    From Wikipedia, the free encyclopedia

    In computing, a process is an instance of a computer program
    that is being executed. It contains the program code and its current activity. Depending on the operating system (OS), a process may be made up of multiple threads of execution that execute instructions concurrently.

    在计算机中，一个进程是一个正在执行的计算机程序的实例，它包含了这个程序的代码和目前的活动。进程依赖于操作系统，一个进程获取是有多个线程，执行指令组成的。

    Thread (computing)

    From Wikipedia, the free encyclopedia

    Threads differ from traditional multitasking operating system processes in that:

    • processes are typically independent, while threads exist as subsets of a process
    • processes carry considerably more state information than threads, whereas multiple threads within a process share process state as well as memory and other resources
    • processes have separate address spaces, whereas threads share their address space
    • processes interact only through system-provided inter-process communication mechanisms
    • context switching between threads in the same process is typically faster than context switching between processes.

    线程和多任务系统中进程的不同：

    • Process是独立的，而Thread必须依赖与进程(Process)
    • Process比线程(Thread)携带了更多的状态(state)信息，然而多数Thread在一个Process内部共享进程的状态以及内存和其他的资源
    • Process有自己的地址(Address)空间，而线程（Thread）共享他们的地址空间
    • Process只能通过系统提供的交互机制进行交互
    • 在同一进程中，线程之间的上下文转换往往要比进程之间的上下文转换要快
